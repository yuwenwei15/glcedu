# 桂林学院校园新闻聚合平台

## 一、项目客群

本项目面向桂林学院全体师生及关注学校动态的社会人士。主要用户包括：在校学生（了解校园快讯、通知公告、学术动态）、教职工（获取学校要闻和工作通知）、以及关注桂林学院的校友和社会公众（通过新媒体动态了解学校发展）。

## 二、需求分析

### 2.1 背景与问题

桂林学院官网新闻中心信息分散在多个栏目页面，缺乏统一的聚合入口；同时，学校在B站、贴吧等新媒体平台的内容无法与官网新闻统一浏览。用户需要在多个平台间切换才能获取完整的校园资讯。

### 2.2 功能性需求

| 功能模块 | 具体需求 | 优先级 |
|---------|---------|-------|
| 新闻浏览 | 按栏目分类展示新闻列表，支持分页浏览 | 高 |
| 新闻详情 | 展示新闻正文、作者、来源、发布时间、阅读量 | 高 |
| 全文搜索 | 按标题和摘要关键词搜索新闻 | 高 |
| 官网爬取 | 定时从桂林学院官网抓取各栏目新闻并入库 | 高 |
| 新媒体聚合 | 抓取B站视频和百度贴吧帖子中与桂林学院相关的内容 | 中 |
| AI总结 | 调用大语言模型为新闻生成150字以内的智能摘要 | 中 |
| 图片代理 | 代理外站封面图片（解决B站CDN防盗链问题） | 中 |
| 管理鉴权 | 管理接口需携带Token认证，防止未授权操作 | 中 |
| 手动触发 | 提供管理接口手动触发爬取任务 | 低 |
| 全量回填 | 支持一次性抓取某栏目或全站的全部历史新闻 | 低 |
| 接口限流 | 对AI总结等高成本接口按IP限流，防止被脚本刷爆 | 低 |

### 2.3 非功能性需求

- **响应式设计**：适配桌面端和移动端浏览器，移动端提供汉堡菜单导航。
- **性能**：AI总结结果缓存到数据库，避免重复调用消耗token；图片代理设置7天缓存；首页查询使用聚合优化避免N+1问题；数据库表建立联合索引加速分页查询；**前端 Tailwind 改为本地构建并 purge（约29KB），替代 Play CDN 的浏览器端现编译，消除首屏延迟与样式闪烁**；**MySQL 连接池启用 `pool_recycle`/`pool_pre_ping`，避免空闲后“server has gone away”**；**滚动监听用 `requestAnimationFrame` 节流，避免卡顿**；**静态资源按文件修改时间追加版本号，避免回头客吃到旧缓存**；字体请求前置 `preconnect` 加速握手。
- **安全性**：管理接口采用Token鉴权机制，未授权请求返回401；**管理令牌仅经请求头 `X-Admin-Token` 传递，不接受URL参数，避免泄露到访问日志/浏览器历史**；**生产环境（`APP_ENV=production`）下若沿用默认弱密钥则拒绝启动**；**AI总结接口按客户端IP限流（每分钟5次）防刷烧token**；**首页引导抓取改用免鉴权专用接口，仅在数据库为空时可触发，前端不再硬编码管理令牌**；图片代理仅允许白名单域名，防止成为开放代理；外链使用`rel="noopener"`。
- **数据一致性**：阅读量计数采用SQL原子操作，避免并发竞态导致计数丢失。
- **部署健壮性**：定时调度器支持多worker部署时仅在单进程启用（`ENABLE_SCHEDULER`），并规避开发模式 reloader 的重复启动，避免定时任务重复执行。
- **可用性**：首页空数据时提供一键抓取按钮，降低初次使用门槛；404/500自定义错误页面提供友好提示与返回导航。

### 2.4 系统用例

主要用例流程：

1. **浏览新闻**：用户访问首页 → 查看焦点新闻轮播 → 点击进入栏目 → 分页浏览 → 点击查看详情
2. **搜索新闻**：用户输入关键词 → 系统按标题/摘要模糊匹配 → 展示分页结果
3. **AI总结**：用户在文章详情页点击"生成AI总结" → 系统调用LLM生成摘要 → 缓存并展示
4. **自动爬取**：系统每日6:00定时触发官网爬取，6:30触发B站/贴吧爬取
5. **管理操作**：管理员携带Token调用管理接口 → 系统验证Token → 执行爬取/管理任务

### 3.5 架构说明

本项目采用 MVC 分层架构：
- **Model层**：`app/models.py` 定义 Category、Article、ScrapeLog 三个数据模型
- **View层**：`app/templates/` 下的 Jinja2 模板负责页面渲染
- **Controller层**：`app/views.py`（页面路由）+ `app/api/news.py`（RESTful API）

独立模块：
- `app/scraper/`：爬虫模块（官网爬虫 + MediaCrawler适配器 + 定时调度）
- `app/ai/`：AI总结服务模块
- `app/api/auth.py`：管理接口鉴权装饰器
- `app/api/ratelimit.py`：进程内接口限流（滑动窗口，保护高成本接口）
- `tests/`：自动化测试套件
- `tailwind.config.js` / `package.json`：前端样式构建配置（Tailwind 本地编译）

### 5.1 首页

（请截图：首页完整页面）

### 5.2 栏目列表页

点击任一新闻栏目进入列表页，左侧显示封面缩略图，右侧显示标题、发布日期、摘要。支持分页浏览，每页15条。

（请截图：某一栏目的列表页面）

### 5.3 新闻详情页

展示新闻完整正文，包含面包屑导航、标题、发布时间/来源/作者/阅读量元信息。页面中部提供AI总结卡片，点击按钮可生成智能摘要。底部展示相关文章推荐。

（请截图：文章详情页 + AI总结生成效果）

### 5.4 搜索功能

顶部导航栏提供搜索入口，输入关键词后展示匹配结果列表，支持分页。

（请截图：搜索结果页面）

### 5.5 AI新闻总结

在文章详情页点击"生成AI总结"按钮，系统调用大语言模型（Qwen3-Coder-30B）生成150字以内的客观摘要，结果缓存到数据库，下次访问直接展示。

（请截图：AI总结生成前后对比）

### 5.6 新媒体内容聚合

系统通过MediaCrawler工具抓取B站中搜索"桂林学院"的视频信息和百度贴吧"桂林学院吧"的帖子内容，统一入库展示。B站内容展示视频封面和简介并提供跳转链接，贴吧内容展示帖子正文。

（请截图：新媒体动态栏目 和/或 贴吧讨论栏目）

### 5.7 管理接口鉴权

管理类API（如全量爬取、社交媒体抓取）需在请求头中携带 `X-Admin-Token`，令牌**仅经请求头传递，不接受URL参数**，避免泄露到访问日志或浏览器历史；未授权请求返回401错误。生产环境（`APP_ENV=production`）下，若 `SECRET_KEY`/`ADMIN_TOKEN` 仍为默认弱值，应用将拒绝启动，防止弱口令上线。

首页空数据时的"一键抓取"按钮改用**专用的免鉴权引导接口**：仅在数据库为空时可触发，有数据后自动返回403停用，并对触发频率限流，前端不再硬编码管理令牌。

此外，AI总结接口对真正触发生成的请求按客户端IP限流（每分钟5次），防止被脚本循环调用刷爆、消耗token。

（请截图：无Token调用管理接口返回401的示例，如Postman截图）

### 5.8 自定义错误页面

当用户访问不存在的页面或服务器发生内部错误时，系统展示自定义的404/500错误页面，提供友好提示和返回首页的导航链接，而非默认的白底报错页。

（请截图：404页面效果）

页面在移动端自动适配，导航栏切换为汉堡菜单，内容区域改为单列布局。

### 7.4 代码质量保障

- **自动化测试**：pytest套件包含25个测试用例，覆盖API、爬虫、页面三层，使用内存SQLite数据库隔离测试环境
